Overview

Design Philosophy

The Celant-Redux document automation engine is designed with following design goals:

  1. Professional User Base
  2. Actual Document as Centerpiece
  3. High Performance
  4. Maximize User Experience

Professional Users

From the very first project you work on, you will immediately know that this app was designed for the Pros. This is because the default view of the project is the "redline" mode, that lawyers, bankers and CEOs will be used to, which show the changes from the form template. The app also provides short descriptions on the left panel for choices to be made, and it typically contains jargons and industry specific language, which gets the point across succinctly to experienced users. To assist with the learning curve, Help section on the right panel provides additional information to help Users with the decision making.

Actual Document is Centerpiece

Most of the document assembly engine available on the market will hide the underlying document from the Users. This is because they are designed to shield the User from the complex underlying legal document and to prevent overwhelming them. Instead, they focus on the "questionnaire" or the "interview" screen, in which the Users are asked to answer them. It may work well for beginners who have no clue, but for professionals, how the underlying document changes is of upmost importance. And since this is a platform for the pros, the underlying document is the centerpiece, literally, in our app. And the changes are shown in the redline mode (which can be turned off), so that the User can see how the document is changing from the template form.

High Performance

Any change you make on the document will be reflected immediately and instantaneously. This is due to the magic of modern day web-browser. This is also because in our design, we removed the slowest part of the application, which is, for virtually all other document assembly software, Microsoft Word. We walked away from being a Microsoft Word Add-on, because it also meant a fresh start with no baggage. Free from massive overhead of Microsoft Word, our app is agile and nimble, and will respond and react to your commands immediately and instantenously.

Maximize User Experience

Our app is designed to be fun to use. It is highly intuitive to use, and promotes user experimentation. What are some tools to make this happen? First, our Smart Scrolling feature will automatically scroll to the part of the document that are changing due to the user input. If there are more than 1 instances of change, then you can navigate them through Smart Navigation buttons.

It respects your time, and handles multiple documents at once. Once a variable has been put in, it will populate it throughout all of the Projects.

We permit Users to designate a level of certainty to each variable, and such levels are color coordinated: Green means certain whereas Red means uncertain. And you can select to show only Uncertain items, etc. There are also "undo" and "redo" buttons to easily revert back the changes you have made.
